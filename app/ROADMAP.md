# Roadmap — แคมเปญเขย่าแคปซูล (ร้านขายยา)

เอกสารนี้สรุปทิศทางงานแบบ **Now / Next / Later** อ้างอิงสถานะปัจจุบันของโค้ดและ `README.md` ปรับเติมตามวันจริงของร้านและทีมได้

## สถานะอ้างอิง (สั้น ๆ)

- **สแตก:** Vite + React, API (`server/`, `api/`), DB production ผ่าน Neon, local/test ใช้ SQLite
- **ฟีเจอร์หลักที่มีในโค้ด:** ลงทะเบียน, สุ่มรางวัล, wallet, แชร์/LIFF, redeem พนักงาน, หน้า admin
- **คุณภาพ:** มี `npm test` ครอบ flow หลัก — รักษาให้ผ่ก่อนปล่อยทุกครั้ง

---

## Now (ก่อน / ระหว่างเปิดแคมเปญจริง)

- [ ] ยืนยันค่า env บน production (Vercel หรือ host ที่ใช้): `DATABASE_URL`, `VITE_LIFF_ID`, LINE ที่เกี่ยวข้อง, `ADMIN_KEY` / `STAFF_REDEEM_PIN` ตามนโยบายร้าน
- [ ] รัน `npm run build` และ `npm run e2e` บน environment ใกล้ production
- [ ] ทดสอบบนมือถือจริง: เปิดจาก LINE → LIFF → ลงทะเบียน → เล่นเกม → รับรางวัล → แสดง wallet
- [ ] กำหนดวิธี backup / export ข้อมูลลูกค้าและรางวัล (ถ้าร้านต้องใช้รายงาน)

## Next (หลัง launch 1–2 รอบ)

- [ ] รายงานสรุปให้ร้าน (export CSV หรือ dashboard เพิ่มใน admin) — ออกแบบตามที่ร้านต้องใช้จริง
- [ ] ปรับ copy/รูปแคมเปญ (`server/campaign.mjs` + สินทรัพย์ UI) โดยไม่แตะ flow หลัก
- [ ] ติดตาม error rate / performance (เช่น log, health `/health`) ถ้า host รองรับ

## Later (ตามความต้องการธุรกิจ)

- [ ] แยกเนื้อหาแคมเปญออกจากโค้ด (CMS หรือ config กลาง) ถ้ามีหลายร้านหรือหลายรอบแคมเปญ
- [ ] LINE Messaging API / แจ้งเตือน (rich message, ย้ำสิทธิ์) — นอกขอบ LIFF ปัจจุบัน
- [ ] บทบาท admin ละเอียด (หลายคน, audit log) หลังมีข้อกำหนดชัด
